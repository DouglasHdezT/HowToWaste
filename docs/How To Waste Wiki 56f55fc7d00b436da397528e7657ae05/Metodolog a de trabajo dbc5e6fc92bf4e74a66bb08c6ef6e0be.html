Metodología de trabajo

Para el trabajo en equipo, se implementó la técnica llamada Pair Programming, esta técnica es implementada para el desarrollo ágil de software. Esta técnica consiste en que dos programadores, ambos del mismo equipo, están trabajando de manera conjunta, un programador está como conductor y otro está escribiendo código.

¿Cómo se implementó está técnica de desarrollo ágil en How To Waste?

Cada miembro del equipo, contó con un rol y una tarea específica, es decir, la tarea se convirtió en tareas modularizar, en la cual un programador escribía código y otro programador era conductor, así mismo, investigador. Al momento de implementar esta técnica en el desarrollo del proyecto, se pudo observar un avance significativo, es decir, se logró cumplir con los requerimientos y que todo el equipo trabaje de manera conjunta.

"Divide y vencerás"

Github y contribución comunitaria.

Por medio de una serie de pull request, el equipo pudo trabajar de una manera estandarizada, pequeños módulos para la aplicación. API y Cliente.

En github, cada branch se manejó con un nombre especificado para cada uno de los métodos a desarrollar.

Así mismo, se logró estandarizar mensajes de commits, para poder tener un mejor enfoque de lo que se estaba manejando en el código.

git commit -m "Add: Agregar método no existente"
git commit -m "Update: Actualizar método existente"
git commit -m "Fix: Remendar error luego de probar"
git commit -m "Test: Probar nuevo método o el cambio en mpetodo exdistente"
git commit -m "Delete: Quitar agregardos o código de más"
